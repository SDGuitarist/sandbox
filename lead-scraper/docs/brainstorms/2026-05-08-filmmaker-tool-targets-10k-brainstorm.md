# Brainstorm: Scale Lead Scraper to 10K via Filmmaker Tool Communities

**Date:** 2026-05-08
**Status:** Draft
**Trigger:** Tony Amat meeting (May 7) -- impressed by lead scraper, wants 10x volume (1,500 -> 10,000+) for October workshop at Spark Studios.

---

## What We're Building

Expand the lead scraper's target list from 2 Facebook groups + 16 Instagram hashtags + 14 Eventbrite keywords to a comprehensive set of **filmmaker tool communities** across all three sources. The goal is 10,000+ leads in the pipeline for the October Spark Studios workshop.

Tony specifically named Premiere Pro SD groups and DaVinci Resolve users. This brainstorm maps the full filmmaker toolkit to scrapable communities, then extends that map to Instagram hashtags and Eventbrite event keywords.

---

## Why This Approach

**Current state:** 1,211 leads (738 Instagram, 258 Facebook, 199 Eventbrite, 16 venue). Only 2 Facebook groups configured. The existing Eventbrite keywords and Instagram hashtags are generic creative/musician terms, not filmmaker-tool-specific.

**The gap:** Zero tool-specific communities in the scraper config. Filmmakers self-organize around tools (Premiere Pro user groups, DaVinci Resolve forums, Final Draft writers circles). These communities are high-intent -- members actively use the tools AI is disrupting.

**Why tool communities convert better than general creative groups:**
- Members self-identify by workflow role (editor, colorist, writer, composer)
- The positioning doc has specific language per tier (writers, directors, composers, post)
- Tool users understand AI disruption viscerally -- it touches their daily work
- The workshop messaging has three hooks, not just ethics: urgency, hands-on tools, rights/protection

---

## Key Decisions

### 1. Audience: Filmmakers first, adjacent creatives secondary

Primary targets are indie filmmakers, directors, editors, composers, screenwriters. Secondary targets (YouTubers, wedding videographers, podcasters, corporate video producers) come from the same tool groups but get lower enrichment priority.

The enrichment pipeline classifies leads into segments. Filmmaker segment gets outreach first. Adjacent creatives get outreach if capacity allows.

### 2. Volume-first across all tools

Not prioritizing by positioning tier (writers > composers > post). Scrape all tool communities simultaneously and let the enrichment pipeline classify. Fastest path to 10K.

### 3. SD-specific groups only (Facebook)

No national/global groups. Lesson learned: Jamie Lee Kendall (SF resident) was scraped from an SD filmmaking group, passed quality filters, wasted a high-value outreach slot. SD-specific groups avoid this. Trade-off: smaller pool per group (50-500 members), but nearly 100% local.

### 4. All three sources: Facebook + Instagram + Eventbrite

Facebook groups for community members. Instagram hashtags for active creators posting their work. Eventbrite keywords for people attending tool-related events in SD. All three sources feed the same enrichment pipeline.

## Implementation Approach

### Phase 1: Research and Validate Targets (Manual)

Before adding anything to config.py, manually verify:
- **Facebook:** Each group exists, is active, and is SD-specific. Many tool-specific SD groups won't exist. That's expected. General filmmaker groups carry the volume.
- **Instagram:** Each hashtag has meaningful post volume (50+ posts). Invented hashtags like `#PremiereProSD` likely have zero posts. Check before adding.
- **Eventbrite:** Each keyword returns current or recent SD events. Niche terms may return nothing.

**Output:** A validated list of real Facebook group URLs, confirmed Instagram hashtags with post volume, and Eventbrite keywords that return SD events. Split into "confirmed" and "doesn't exist" so we know the real target count.

### Phase 2: Add to Config

Append validated group URLs to `config.py` SOURCES dict. Add new Instagram hashtags and Eventbrite keywords. No code changes needed for Facebook (just URL list). Instagram and Eventbrite may need new keyword entries.

### Phase 3: Run and Monitor

Run scraper with expanded targets. Monitor Apify costs (each source = actor runs). Check for duplicate leads across sources. Verify location signal on Facebook leads.

## Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| Many SD-specific tool groups don't exist | Fall back to general filmmaker groups + broader "San Diego Filmmakers" communities |
| Apify costs spike with 30+ groups | Batch scrapes across multiple days. Monitor cost per lead. |
| Duplicate leads across sources | Handled. `run.py dedup` merges by email or name match cross-source. Run after each scrape cycle. |
| Location false positives from Facebook | Enrichment pipeline has location verification step. Flag but don't skip. |
| Low-quality adjacent leads dilute pipeline | Enrichment classifies into segments. Filmmaker segment gets priority outreach. |

## Resolved Questions

1. **Apify cost budget:** $10-30/cycle is fine. Can run all groups in one cycle without batching across days.
2. **Cross-source dedup:** Already handled. `run.py dedup` finds duplicates by email or name match (case-insensitive) and merges them. Cross-source duplicates will be caught.
3. **Scrape frequency:** One-time bulk scrape. Run once, load the pipeline, focus on enrichment and outreach. Revisit for recurring if volume is still short.
